XAIP Playground — Trust Before Agent Delegation

A read-only browser demo for seeing how historical trust evidence can change an agent's tool choice.

AI agents increasingly choose tools, services, and counterparties on your behalf. Today, many of those choices are blind: the agent sees a name and a description, but not much evidence about how that option has behaved before.

XAIP adds behavior-derived trust evidence before delegation. This playground shows the basic idea without posting receipts, creating keys, or changing any live score.

Read-only demo. This page does not post receipts, does not generate wallet or signing keys, and does not call the aggregator receipt endpoint. Trust scores are behavior-derived evidence, not safety guarantees.

Blind vs XAIP Selection

This frontend selection logic is an educational playground simplification. It is not live /v1/select behavior and does not change routing, scoring, or production decisions.
